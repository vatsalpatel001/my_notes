What are Constraints in SQL?

Constraints in SQL are rules defined for the data columns in a table. They are used to limit the type of data that can be inserted or updated in a table. Common types of constraints include:

  • NOT NULL: Ensures that a column cannot have a NULL value.
  • UNIQUE: Ensures that all values in a column are different.
  • PRIMARY KEY: A combination of NOT NULL and UNIQUE. Uniquely identifies each record in a table.
  • FOREIGN KEY: Ensures referential integrity between two tables.
  • CHECK: Ensures that all values in a column satisfy certain conditions.
  • DEFAULT: Provides a default value for a column when none is specified.
What is a Primary Key?

A primary key is a constraint in SQL that uniquely identifies each record in a table. It must contain unique values, and no NULL values are allowed. A primary key is implemented using the PRIMARY KEY constraint. Typically, it is applied to one or more columns of a table to ensure the uniqueness of each record.

What is a UNIQUE constraint?

The UNIQUE constraint in SQL ensures that all values in a column are distinct (unique). It is similar to the PRIMARY KEY constraint but allows for NULL values in the column (excluding the unique constraint).

What is a Foreign Key?

A Foreign Key is a column or a set of columns in a table that refers to the primary key of another table. It establishes a link between the two tables, creating a relationship. The FOREIGN KEY constraint is used to enforce referential integrity between the two tables.

What is a Join? List its different types.

In SQL, a join is used to combine rows from two or more tables based on a related column between them.

  • INNER JOIN: Returns only the rows where there is a match in both tables. 
    SELECT employees.employee_id, employees.employee_name, departments.department_name
                            FROM employees
                            INNER JOIN departments ON employees.department_id = departments.department_id;
  • LEFT (OUTER) JOIN: Returns all rows from the left table and matched rows from the right table. 
    SELECT customers.customer_id, orders.order_id, orders.order_date
                            FROM customers
                            LEFT JOIN orders ON customers.customer_id = orders.customer_id;
  • RIGHT (OUTER) JOIN: Returns all rows from the right table and matched rows from the left table. 
    SELECT orders.order_id, orders.order_date, customers.customer_name
                            FROM orders
                            RIGHT JOIN customers ON orders.customer_id = customers.customer_id;
  • FULL (OUTER) JOIN: Returns all rows when there is a match in either table. 
    SELECT employees.employee_id, employees.employee_name, departments.department_name
                            FROM employees
                            FULL JOIN departments ON employees.department_id = departments.department_id;
  • CROSS JOIN: Returns the Cartesian product of the two tables (all possible combinations). 
    SELECT employees.employee_name, departments.department_name
                            FROM employees
                            CROSS JOIN departments;
  • SELF JOIN: Joins a table with itself. 
    SELECT e1.employee_name AS employee1, e2.employee_name AS employee2
                            FROM employees e1
                            JOIN employees e2 ON e1.manager_id = e2.employee_id;
What is a Self-Join?

A Self-Join is a specific case of a SQL join where a table is joined with itself. It is used when a table has a hierarchical structure or when it contains a foreign key that references its own primary key. In a self-join, you need to use table aliases to differentiate between the two instances of the same table.

What is a Cross-Join?

A Cross-Join, also known as a Cartesian Join, returns the Cartesian product of the two tables involved. In other words, it combines each row from the first table with every row from the second table, resulting in a large result set. Cross-Joins are not commonly used in practice due to their potential to generate a vast number of rows.

What is an Index? Explain its different types.

An Index is a database object that improves the speed of data retrieval operations on a database table. It is created on one or more columns of a table. Different types of indexes include:

  • Clustered Index: Determines the physical order of data rows in a table.
  • Non-Clustered Index: Does not alter the physical order of the table, and it uses a separate structure to store index data.
  • Unique Index: Ensures that the indexed columns contain unique values.
  • Composite Index: Created on multiple columns to improve query performance for conditions involving those columns.
What is the difference between Clustered and Non-clustered index?

The main difference between a Clustered and Non-clustered index lies in the way they store and retrieve data:

  • Clustered Index: Determines the physical order of data rows in a table. The table can have only one clustered index, and the data rows are sorted based on the key values. It directly affects the order of the data on disk.
  • Non-Clustered Index: Does not alter the physical order of the table. It uses a separate structure to store index data, and the data rows are stored in a different order. A table can have multiple non-clustered indexes.
What is Data Integrity?

Data Integrity in a database ensures the accuracy, consistency, and reliability of data. It involves the application of rules and constraints to maintain the quality of data, preventing issues such as duplicate records, invalid entries, and data inconsistencies. Data integrity is enforced through mechanisms like constraints, relationships, and validation rules.

What is a Query?

A Query in the context of databases refers to a request for information from a database. It is a command or set of commands used to retrieve, insert, update, or delete data in a database. SQL (Structured Query Language) is a standard language for querying relational databases, and it provides various statements for performing different operations on databases.

What is a Subquery? What are its types?

A Subquery, also known as a nested query or inner query, is a query embedded within another query. It is used to retrieve data that will be used by the main query. Types of subqueries include:

  • Scalar Subquery: A subquery that returns a single value.
  • Row Subquery: A subquery that returns one or more rows of data.
  • Table Subquery: A subquery that returns a table.
What is the SELECT statement?

The SELECT statement in SQL is used to query the database and retrieve data from one or more tables. It allows you to specify the columns you want to retrieve, the tables you want to query, and conditions for selecting specific rows. The basic syntax of the SELECT statement is:

SELECT column1, column2, ...
        FROM table_name
        WHERE condition;
What are some common clauses used with SELECT query in SQL?

Common clauses used with the SELECT query in SQL include:

  • WHERE: Specifies conditions for filtering rows.
  • ORDER BY: Sorts the result set based on one or more columns.
  • GROUP BY: Groups rows that have the same values in specified columns.
  • HAVING: Applies conditions to grouped rows after the GROUP BY clause.
  • JOIN: Combines rows from two or more tables based on a related column.
  • LIMIT: Limits the number of rows returned by the query.
  • OFFSET: Skips a specified number of rows before starting to return rows.
What are UNION, MINUS, and INTERSECT commands?

In SQL, UNION, MINUS, and INTERSECT are set operations:

  • UNION: Combines the result sets of two or more SELECT statements, eliminating duplicate rows.
  • MINUS (or EXCEPT): Returns the rows that are present in the first result set but not in the second result set, eliminating duplicates.
  • INTERSECT: Returns the common rows between two result sets, eliminating duplicates.
What is a Cursor? How to use a Cursor?

In SQL, a cursor is a database object used to traverse the result set of a query. Cursors provide a way to iterate over the rows of a result set one at a time.

Usage of a cursor involves the following steps:

  1. Declare a cursor.
  2. Open the cursor.
  3. Fetch rows from the cursor one by one or in a bulk.
  4. Close the cursor when done.

Example:

DECLARE cursor_name CURSOR FOR
        SELECT column1, column2
        FROM your_table;
        
        OPEN cursor_name;
        
        FETCH NEXT FROM cursor_name;
        
        -- Process the fetched row(s)
        
        CLOSE cursor_name;
What are Entities and Relationships?

In the context of databases, entities are objects or concepts represented by tables. Relationships define associations between these entities.

List the different types of relationships in SQL.

Common types of relationships in SQL include:

  • One-to-One (1:1): Each row in the related table is related to only one row in the main table.
  • One-to-Many (1:N): Each row in the main table can be related to multiple rows in the related table.
  • Many-to-One (M:1): Multiple rows in the main table can be related to one row in the related table.
  • Many-to-Many (M:N): Multiple rows in both tables can be related to each other.
What is an Alias in SQL?

In SQL, an alias is a temporary name assigned to a table or column for the purpose of a query. It is used to make the column or table names more readable or to provide a different name.

Example:

SELECT column1 AS alias_name
        FROM your_table;
What is a View?

In SQL, a view is a virtual table based on the result of a SELECT query. It allows users to query and manipulate data as if it were a regular table, but the data is not stored directly in the view.

What is Normalization?

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves breaking down large tables into smaller ones and defining relationships between them.

What is Denormalization?

Denormalization is the process of intentionally introducing redundancy into a database by combining tables. This is done to improve query performance at the cost of some redundancy and potential update anomalies.

What is OLTP?

OLTP (Online Transaction Processing) is a type of data processing that manages transaction-oriented applications. It is characterized by a large number of short online transactions, such as INSERT, UPDATE, and DELETE operations. OLTP databases are designed for real-time data processing and are optimized for quick and accurate transaction processing.

What are the differences between OLTP and OLAP?

OLAP (Online Analytical Processing) is different from OLTP in terms of:

  • Purpose: OLTP is used for transactional processing, while OLAP is used for analytical processing and reporting.
  • Database Design: OLTP databases are normalized, and OLAP databases are often denormalized for faster query performance.
  • Operations: OLTP involves simple CRUD operations, while OLAP involves complex queries and aggregations.
  • Data Freshness: OLTP requires real-time data, while OLAP deals with historical data and is not concerned with real-time updates.
  • Concurrency: OLTP systems handle a high level of concurrent transactions, while OLAP systems are typically read-intensive with fewer concurrent updates.
What is Collation? What are the different types of Collation Sensitivity?

Collation refers to the set of rules that dictate how characters are sorted and compared in a database. Different types of collation sensitivity include:

  • Case Sensitivity: Determines whether uppercase and lowercase letters are treated as distinct. Examples include case-sensitive and case-insensitive collations.
  • Accent Sensitivity: Determines whether accented characters are treated as distinct from their non-accented counterparts.
  • Kana Sensitivity: Relevant in Japanese collations and determines whether Kana characters are treated as distinct from their Romanized counterparts.
  • Width Sensitivity: Determines whether full-width and half-width characters are treated as distinct.
What is a Stored Procedure?

A stored procedure is a precompiled collection of one or more SQL statements that can be executed as a single unit. Stored procedures are typically stored in a database and can be called by applications or other stored procedures. They help improve performance, code modularity, and security by encapsulating complex logic and business rules.

What is a Recursive Stored Procedure?

A recursive stored procedure is a stored procedure that calls itself. It is used when a task can be broken down into subtasks that are similar to the original task. Recursive stored procedures are often used in hierarchical data structures such as trees and graphs.

How to create empty tables with the same structure as another table?

To create an empty table with the same structure as another table in SQL, you can use the following syntax:

CREATE TABLE new_table AS SELECT * FROM existing_table WHERE 1=0;

This creates a new table called new_table with the same structure as existing_table but without copying any data.

What is Pattern Matching in SQL?

Pattern matching in SQL involves searching for a specified pattern in a column. Commonly used patterns include:

  • %: Represents zero or more characters.
  • _: Represents a single character.
  • [character_list]: Represents any single character in the list.
  • [^character_list]: Represents any single character not in the list.

Pattern matching is often used in conjunction with the LIKE operator in SQL queries.
